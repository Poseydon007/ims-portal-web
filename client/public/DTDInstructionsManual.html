📋 What is the DTD System?

The Day-to-Day Document (DTD) system is True East Mining's official numbering and registration system for all operational documents produced during daily business — letters, reports, purchase orders, tenders, invoices, drawings, meeting minutes, and more.

Every document produced by True East that leaves your desk, goes to a client, vendor, government entity, or gets filed officially must be registered in the DTD Register and carry a unique DTD code.

ℹ️
DTD vs IMS: The IMS Library (TE-IMS) contains controlled templates, policies, and procedures. The DTD system (TE-DTD) is for the operational documents you produce daily using those templates. Think of IMS as the rulebook, DTD as the game.

⚡ Quick Reference

Need to register a document fast? Here's the 60-second version:

Open the DTD Register

Go to the shared DTD Register (Excel/Google Sheets). Find the tab matching your document type.

Find your department's section

Scroll to your department. Codes are grouped by department within each tab.

Claim the next available code

Find the first row with Status = Available. Change it to Draft.

Fill in your details

Enter: Document Title, your name as Originator, and today's date as Date Claimed.

Use the DTD code on your document

The code in column B is now yours. Put it in the document header/filename immediately.

Update when issued

When the document is approved and sent, change Status to Issued. Fill in Date Issued, Recipient, and Delivery Method.

🔤 Code Structure

Every DTD code follows this exact structure:

Format: TE-DTD-[Type]-[Dept]-[Year]-[Seq] Example: TE-DTD-LTR-COMM-2026-003 Meaning: True East | Day-to-Day | Letter | Commercial Dept | Year 2026 | 3rd in sequence
ComponentWhat It MeansExample
TE-DTDCompany prefix + Day-to-Day marker (always fixed)TE-DTD
[Type]Document type code — 3 letters from the 16 typesLTR, PO, RPT
[Dept]Department code — the team producing the documentCOMM, ENG, FIN
[Year]Calendar year of registration (4 digits)2026
[Seq]Sequential number within the type tab (3 digits)001, 025, 103

✏️ How to Register a Document

Before You Start

You must register your document before you create it. The DTD code needs to exist in the register first, then goes on the document. Never create the document first and try to get a number later.

Step-by-Step Process

Identify your document type

What are you producing? A letter? A report? A purchase order? Match it to one of the 16 type codes (see the Type Codes reference below).

Open the correct tab in the DTD Register

Each document type has its own tab. Go to the tab matching your type code (e.g., "LTR" tab for letters, "PO" tab for purchase orders).

Find your department's block

Within the tab, codes are grouped by department. Scroll to your department's section. The Department column tells you where you are.

Claim the next available row

Find the first row where Status = Available. Change the Status to Draft.

Fill in the required fields

Document Title: Clear, descriptive title (e.g., "Cover Letter — Ma'aden Proposal Submission")
Originator: Your full name
Date Claimed: Today's date

Use the DTD code immediately

Copy the code from column B (e.g., TE-DTD-LTR-COMM-2026-003) and place it in your document header and filename.

When issued, update the register

Once the document is approved, signed, and sent:
— Change Status to Issued
— Fill in Date Issued, Recipient, and Delivery Method

⚠️
First come, first served. If you claim a code, it's yours. If someone else got to it first, take the next one. Never overwrite someone else's claimed row.

🔄 Document Lifecycle

Every DTD code goes through a defined lifecycle. The status in the register must always reflect the current state:

Available → Draft → Issued

Normal flow: pre-allocated → claimed for drafting → approved and sent

Special Statuses

StatusMeaningWhen to Use
AvailablePre-allocated, not yet claimedDefault state — don't touch unless you're claiming it
DraftClaimed, document in preparationThe moment you decide to produce this document
IssuedDocument approved, signed, and sentAfter final approval and delivery to recipient
VoidedCancelled before issuanceYou claimed it but the document is no longer needed — add a note explaining why
SupersededReplaced by a newer versionA new document replaces this one — note the new DTD code
🚫
Never go backwards. You cannot change Issued back to Draft, or Voided back to Available. Once a status advances, it stays. If you need a new version, claim a new code.

⚖️ Golden Rules

1️⃣
Register BEFORE you create. The number must exist in the register before it appears on any document. No exceptions.
2️⃣
Never reuse a number. Every DTD code is used exactly once, forever. Voided codes stay voided. Superseded codes stay superseded.
3️⃣
Never delete a row. The register is a permanent record. If something is wrong, void it and add a note — don't delete it.
4️⃣
One code = one document. Don't share a DTD code across multiple documents. Each document gets its own unique code.
5️⃣
Update status promptly. When a document moves from Draft to Issued, update the register the same day. Don't let it go stale.
6️⃣
Use the right type and department. A purchase order is a PO, not a LTR. An engineering report from Engineering uses ENG, not COMM. If unsure, ask the IMS Lead.

❌ Common Mistakes to Avoid

MistakeWhy It's WrongDo This Instead
Creating the document first, then getting a numberThe document may already be sent without a controlled codeRegister first, create second
Using someone else's DTD codeThat code is permanently assigned to their documentClaim your own next available code
Deleting a row because you made an errorBreaks the audit trail — auditors need to see every numberSet status to Voided and add a note
Issuing a PO under LTR because "it's a letter to the vendor"A PO is a purchase order regardless of formatUse the correct document type code
Not updating status after sendingRegister shows Draft but document is already with the clientUpdate to Issued with date and recipient
Using the same code for a revised versionEach version is a separate document in DTDClaim a new code, mark the old one Superseded

❓ Frequently Asked Questions

What if all codes for my department/type are used up?
Contact the IMS Lead. They will add more pre-allocated codes to your section following the next available sequence number. The system is designed to grow.
My department isn't listed on the tab I need. What do I do?
Contact the IMS Lead. If there's a legitimate business reason for your department to produce that document type, they will add your department to that tab. The initial allocation covers the most common combinations, but edge cases happen.
I claimed a code but the document was cancelled. What now?
Change the status to Voided and add a note in the Notes column explaining why (e.g., "Project cancelled by client" or "Superseded by revised scope"). Never delete the row or reuse the number.
Can I use the DTD system for internal draft documents?
The DTD system is for documents that will be officially issued — to clients, vendors, government, or filed as official company records. Informal working drafts, personal notes, and scratch calculations don't need DTD codes. If in doubt, ask: "Will this document leave my desk or become an official record?" If yes, register it.
Do emails need DTD codes?
Regular emails do not. However, if an email serves as an official letter or transmittal (e.g., a formal submission email to a client), consider creating a proper letter (LTR) or transmittal (TMT) document with a DTD code and attaching it to the email.
What happens at the start of a new year?
The IMS Lead will create a new set of pre-allocated codes with the new year (e.g., 2027). Sequence numbers reset to 001. The previous year's register is archived but remains accessible for reference.
Where do I find the DTD Register?
The register is maintained as a shared file (Google Sheets or SharePoint) accessible to all relevant staff. Ask your department head or the IMS Lead for the link. Bookmark it — you'll use it regularly.
Who is the IMS Lead?
The IMS Lead is the person responsible for maintaining the Integrated Management System, including the DTD Register. They manage the register, add codes, resolve disputes, and conduct periodic audits. Contact details are in the Support section below.
