🎤 Trainer Note

Key Point

What is the DTD System?

DTD stands for Day-to-Day. It's True East Mining's official system for numbering, registering, and tracking every operational document the company produces — letters, reports, purchase orders, tenders, invoices, drawings, and more.

Every document that leaves your desk, goes to a client or vendor, or becomes an official company record must carry a DTD code.

Key Point

DTD vs IMS — Know the Difference

  • IMS (TE-IMS) = The rulebook. Policies, procedures, templates, SOPs. These are controlled library documents that rarely change.
  • DTD (TE-DTD) = The game. Operational documents you produce every day using those templates. Letters you write, POs you issue, reports you submit.

Think of it this way: the IMS tells you how to write a letter. The DTD system gives that letter an official number so it can be tracked forever.

Key Point

Why Does This Matter?

  • Traceability: Every document can be found by its unique code — in 5 minutes, not 5 hours
  • Accountability: We know who created what, when, and who received it
  • Audit readiness: ISO auditors and KSA regulators can verify our document trail
  • Professionalism: Clients and partners see an organized, controlled company
  • Legal protection: Numbered, dated documents are evidence — unnumbered files are just noise

How to Register a Document — Live Demo

🎤 Trainer Note

Live Demo Script

Scenario: You need to send a letter to a client

Situation: The Commercial team needs to send a cover letter to Ma'aden with a proposal submission.

Step 1 of 7

Open the DTD Register

Go to the shared register link (Google Sheets / SharePoint). Bookmark it if you haven't already.

Step 2 of 7

Go to the LTR tab

This is a letter, so we use the LTR tab. Click on it at the bottom of the spreadsheet.

Step 3 of 7

Find the Commercial department section

Scroll down to the COMM (Commercial) section. Codes are grouped by department.

Step 4 of 7

Find the next available row

Look for Available in the Status column. That's your row.

Step 5 of 7

Claim it — change Status to Draft

Click the Status cell. Use the dropdown to select Draft. The code is now yours.

Step 6 of 7

Fill in your details

  • Document Title: "Cover Letter — Ma'aden Proposal Submission"
  • Originator: Your name
  • Date Claimed: Today's date
Step 7 of 7

Use the code on your document

Copy the DTD code from column B (e.g., TE-DTD-LTR-COMM-2026-003). Put it in the document header and filename.

Filename: TE-DTD-LTR-COMM-2026-003 - Cover Letter Ma'aden Proposal.pdf

Lifecycle, Rules & Common Mistakes

⏱ 10 minutes📖 Lecture + Scenarios
▼
Lifecycle Flow

How a DTD Code Lives

Available → Draft → Issued

Special exits:

Draft → Voided (cancelled before sending)
Issued → Superseded (replaced by new version)
The 6 Golden Rules

Memorize These

  1. Register BEFORE you create — number first, document second
  2. Never reuse a number — once assigned, it's gone forever
  3. Never delete a row — void it, don't erase it
  4. One code = one document — no sharing codes
  5. Update status promptly — same day when issued
  6. Use the right type code — a PO is a PO, not a letter
🎤 Trainer Note
Present each "What Would You Do?" scenario below. Let the group discuss for 30 seconds before revealing the answer. This is where most learning happens.
🤔 Exercise 3 — What Would You Do?

Scenario A

You claimed code TE-DTD-LTR-COMM-2026-005 for a letter, but the project was cancelled. The letter was never sent. What do you do?

Answer: Set status to Voided. Add note: "Project cancelled — letter not issued." Never delete the row.

Scenario B

You need to send a revised version of a report that was already issued as TE-DTD-RPT-ENG-2026-010. What do you do?

Answer: Claim a NEW code (e.g., TE-DTD-RPT-ENG-2026-011). Set the original to Superseded with a note: "Replaced by TE-DTD-RPT-ENG-2026-011."

Scenario C

Your colleague asks you to use their DTD code on your document because "we're working on the same project." What do you do?

Answer: Refuse. Every document gets its own code. If it's a different document, it needs a different number. Period.

File Naming

How to Name Your File

[DTD Code] - [Title].[ext] TE-DTD-LTR-COMM-2026-003 - Cover Letter Ma'aden Proposal.pdf TE-DTD-PO-SCM-2026-015 - Diamond Bits Supplier ABC.pdf TE-DTD-RPT-GEO-2026-008 - BH-15 Geological Completion.docx
Filing Structure

Where to Save

📁 DTD Documents / 2026 / [Your Dept] / [Type] / [file] Example: DTD Documents / 2026 / COMM / LTR / TE-DTD-LTR-COMM-2026-003 - Cover Letter.pdf
Getting Help

When You're Stuck

  • Instructions Manual: Open the DTD Instructions Manual (HTML) — searchable, always available
  • Quick Guide: TE-IMS-SOP-SYS-001 (one-page SOP)
  • IMS Lead: For adding codes, new department assignments, or unusual cases
  • Your Department Head: For document type questions specific to your work
