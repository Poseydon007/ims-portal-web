<h2 id="s1" class="text-sm font-bold mt-8 mb-3 pb-2" style="color: #081C2E; border-bottom: 2px solid #C49A28;">1. Purpose</h2>

<p class="text-sm leading-relaxed mb-3" style="color: #374151;">This document defines the mandatory and standardised rules for assigning unique identifiers to all controlled documents within True East Mining Company's Integrated Management System (IMS).</p>

<p class="text-sm leading-relaxed mb-2" style="color: #374151;">It ensures:</p>

<ul class="list-none pl-0 space-y-1 mb-3">
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>Every document has a clear, consistent, and instantly meaningful identity.</span></li>
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>Full traceability from legacy/old documents to the new system.</span></li>
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>Easy search, filtering, sorting, and retrieval by users in the field and office.</span></li>
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>No ambiguity for project-specific versus company-wide documents.</span></li>
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>Scalability for future growth (new projects, rigs, departments, document volumes).</span></li>
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>Alignment with ISO 9001, 14001, and 45001 requirements for documented information control.</span></li>
</ul>

<p class="text-sm leading-relaxed" style="color: #374151;">This establishes the baseline logic for the entire True East IMS document database.</p>

<h2 id="s2" class="text-sm font-bold mt-8 mb-3 pb-2" style="color: #081C2E; border-bottom: 2px solid #C49A28;">2. Scope</h2>

<p class="text-sm leading-relaxed mb-2" style="color: #374151;">Applies to all controlled documents managed under the True East IMS, including but not limited to:</p>

<ul class="list-none pl-0 space-y-1 mb-3">
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>Policies, Manuals, Procedures, Plans</span></li>
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>SOPs, Work Instructions</span></li>
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>Forms, Checklists, Registers, Logs</span></li>
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>Drawings</span></li>
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span>References, Appendices, External Standards</span></li>
</ul>

<p class="text-sm leading-relaxed" style="color: #374151;"><strong>Excludes:</strong> Uncontrolled records (emails, informal notes, drafts not yet approved for release).</p>

<h2 id="s3" class="text-sm font-bold mt-8 mb-3 pb-2" style="color: #081C2E; border-bottom: 2px solid #C49A28;">3. Special Case: Treatment of Finance &amp; Accounting Documents</h2>

<p class="text-sm leading-relaxed mb-3" style="color: #374151;">High-level finance policies, procedures, process flows, approval workflows, and non-sensitive general forms/templates are fully included within the IMS and follow the standard TE-IMS numbering and control rules.</p>

<p class="text-sm leading-relaxed" style="color: #374151;">Detailed transactional records, ledgers, invoices, receipts, payroll data, bank reconciliations, tax filings, and any other sensitive or software-generated financial documents are strictly excluded from the main IMS document control. These are managed separately under the Finance Department's dedicated accounting/ERP system. Only summary-level or control-level finance outputs are referenced or attached in the IMS where necessary for oversight.</p>

<h2 id="s8" class="text-sm font-bold mt-8 mb-3 pb-2" style="color: #081C2E; border-bottom: 2px solid #C49A28;">8. Naming Conventions: Document Code vs. File Name</h2>

<p class="text-sm leading-relaxed mb-3" style="color: #374151;">To keep the official Document Code short, mathematically pure, and easy to type in cross-references, the descriptive name of the document is removed from the official code but added to the computer file name.</p>

<ul class="list-none pl-0 space-y-1 mb-3">
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span><strong>Official Code (inside the document header):</strong> <span class="te-code">TE-IMS-FRM-OPS-001_Rev00</span></span></li>
  <li class="flex gap-2 text-sm" style="color: #374151;"><span style="color: #C49A28;">•</span><span><strong>File Name (when saving to SharePoint/Windows):</strong> <span class="te-code">TE-IMS-FRM-OPS-001_Rev00 - Daily Drilling Report.docx</span></span></li>
</ul>

<p class="text-sm leading-relaxed" style="color: #374151;">This ensures that if the title of a document slightly changes in the future, the underlying alphanumeric control code remains intact.</p>
